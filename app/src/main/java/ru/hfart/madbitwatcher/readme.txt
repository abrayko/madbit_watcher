Служба при запуске сразу подключается к DSP и начинает слушать события.

+ Привязка Activity к сервису
 Activity вызывает  HandleService.bindService указывая:
    - флаг, что отвязка после запуска службы не требуется
    - inline реализацию интерфейса ServiceConnection
 HandleService:
    - привязывает сервис
    - получив от сервиса в onServiceConnected IBinder запускает сервис
    - если в вызове HandleService была передана реализация ServiceConnection, то вызывает ее метод onServiceConnected
    - если не указано, то сразу отвязывается от сервиса

 Activity, получив вызов onServiceConnected, регистрирует в сервисе себя, как реализатор интерфейса наблюдателя за событиями от проца DSP

 Интерфейс реализует методы:
    - получение значение громкости
    - получение значения типа входа
    - получение значения частоты дискретизации
    - получение значения пресета
    - получение состояния коннекта к процессору

Обработка событий подключения USB
1. Приложение не запущено
    Права есть -> запуск службы и подключение
    Прав нет -> запрос прав. После получения -> запуск службы и подключение
2. Сервис запущен, окно зарыто
    Права есть -> подключение
    Прав нет -> запрос прав. После получения -> запуск службы и подключение
3. Сервис запущен, окно открыто
    Права есть -> подключение
    Прав нет -> запрос прав. После получения -> запуск службы и подключение
